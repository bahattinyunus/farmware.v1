# Contributing to Farmware

First off, thanks for taking the time to contribute!

The following is a set of guidelines for contributing to Farmware. These are mostly guidelines, not rules. Use your best judgment, and feel free to propose changes to this document in a pull request.

## Code of Conduct

This project and everyone participating in it is governed by the [Farmware Code of Conduct](CODE_OF_CONDUCT.md). By participating, you are expected to uphold this code.

## How Can I Contribute?

### Reporting Bugs

Start by searching GitHub Issues for open and closed problems that match yours. If you can't find it, open a new issue.

### Suggesting Enhancements

If you have an idea for a feature or an enhancement, please open an issue and tag it as an `enhancement`.

### Pull Requests

1.  **Fork the repo** and create your branch from `main`.
2.  **Test your changes** to ensure everything works as expected.
3.  **Make sure your code lints**.
4.  **Issue that pull request!**

## Styleguides

### Git Commit Messages

*   Use the present tense ("Add feature" not "Added feature")
*   Limit the first line to 72 characters or less
*   Reference issues and pull requests liberally after the first line

### Python Styleguide

*   Follow [PEP 8](https://www.python.org/dev/peps/pep-0008/).
*   Use docstrings for functions and classes.
*   Type hinting is encouraged.
